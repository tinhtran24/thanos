# Acceptance Criteria

| # | Criterion | Verification Method |
|---|---|---|
| AC-1 | `model.Project` has optional JSON field `framework`; one detected or overridden value is written to `.thanos/settings.json`, and an empty value omits the key. | `go test ./internal/cli -run 'TestRunInit.*Framework|TestInitHelpIncludesFramework' -v` |
| AC-2 | PHP detection returns `laravel` for exact `laravel/framework` keys in root Composer `require` or `require-dev`, or for regular root files `artisan` plus `bootstrap/app.php`; it returns `wordpress` only when all three root WordPress markers are directories. | `go test ./internal/project -run 'TestDetectFrameworkPHP|TestDetectFrameworkMarkerTypes' -v` |
| AC-3 | TypeScript detection maps exact root `package.json` dependency/devDependency keys to `nextjs`, `nestjs`, `angular`, or `nuxt`, without matching scripts, values, lock files, nested manifests, casing variants, or package-name prefixes. | `go test ./internal/project -run TestDetectFrameworkTypeScript -v` |
| AC-4 | Go detection maps exact root `go.mod` requirements to `gin` or `echo`, including grouped requirements and decimal versioned Echo module paths, while excluding comments, `replace`/`exclude`-only references, source imports, `go.sum`, and similar prefixes. | `go test ./internal/project -run TestDetectFrameworkGo -v` |
| AC-5 | Python detection maps PEP 503-normalized `django`, `flask`, and `fastapi` declarations from the specified PEP 621, Poetry, PEP 735, and root `requirements*.txt` locations, including normal extras/version/marker syntax and excluding unsupported lines and aliases. | `go test ./internal/project -run TestDetectFrameworkPython -v` |
| AC-6 | Rust detection maps `actix-web`, `axum`, and `rocket` from the specified root, workspace, and target Cargo dependency tables, supports renamed dependencies through `package`, normalizes `_` to `-`, and excludes comments, values, lock files, source files, and nested manifests. | `go test ./internal/project -run TestDetectFrameworkRust -v` |
| AC-7 | Detector language selection trims and lowercases the supplied language; unsupported languages and zero matches return `""`; two or more valid matches for the selected language return `""`, including conflicts across separate evidence sources. | `go test ./internal/project -run 'TestDetectFrameworkLanguageSelection|TestDetectFrameworkAmbiguity' -v` |
| AC-8 | Missing and malformed optional evidence is non-fatal and cannot suppress valid evidence from another source; every non-not-found read, root-directory, or marker-stat error is returned with path context. | `go test ./internal/project -run 'TestDetectFrameworkMalformedEvidence|TestDetectFrameworkFilesystemErrors' -v` |
| AC-9 | `runInit` applies the final `--language` before auto-detection. A trimmed non-empty `--framework` overrides detection; whitespace-only `--framework` leaves auto-detection active. Detector errors prevent settings creation. | `go test ./internal/cli -run 'TestRunInit.*Framework' -v` |
| AC-10 | Detection is root-only, read-only, network-free, executes no subprocess, and leaves existing project name/package/build/test/lint detection unchanged. No state-machine or orchestrator files are changed. | `go test ./internal/cli -run 'TestRunInitFrameworkDetectionDoesNotExecuteCommands|TestRunInitDetectsTypeScriptCommandsAndWorkspace' -v && ! rg -n '\"net(/http)?\"|\"os/exec\"' internal/project && git diff --exit-code -- internal/state internal/orchestrator` |
| AC-11 | CLI help contains `--framework`. Both `README.md` and `Technical.md`, independently, document `project.framework`, `--framework`, every canonical value, all supported root evidence categories, final-language selection, ambiguity, omitted-empty behavior, and local/read-only/network-free/no-command guarantees. | `go run ./cmd/thanos help > /tmp/thanos-help.txt && rg -- '--framework' /tmp/thanos-help.txt && go test ./internal/cli -run '^TestFrameworkDocumentation$' -v` |
| AC-12 | Focused detector and CLI tests pass under the unit profile. | `go test ./internal/project -run Framework -v && go test ./internal/cli -run 'Init.*Framework|InitHelpIncludesFramework' -v` |
| AC-13 | The complete repository suite passes with no formatting or module-file inconsistency. | `test -z "$(gofmt -l internal/model/model.go internal/project/project.go internal/project/project_test.go internal/cli/cli.go internal/cli/cli_test.go)" && go mod tidy && git diff --exit-code -- go.mod go.sum && go test ./...` |
